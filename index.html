The one rule that organizes everything: a workflow is a vertical slice — you pick one box per layer, top to bottom. "What should I use?" just means "which slice am I in?"

So… what do I use when?

Answer three questions and your slice falls out.

Is the data sensitive, or do I need it to work offline?
Yes → go Local. VS Code → Cline → localhost → DeepSeek/Qwen → Ollama → your Mac. Nothing leaves the machine, costs nothing per run. No → the cloud is fair game; keep reading.
Is the task hard — real coding, deep reasoning, long agentic work?
Yes → go Frontier. Terminal → Claude Code → Claude. You're paying for quality and the best agent tooling, and it's worth it. No / bulk & repetitive → a cheap open model (DeepSeek) via OpenRouter or local is plenty.
Does it need to run on a schedule or without me watching?
Yes → that's the Orchestration rail — cron, a cloud agent, or a pipeline drives your chosen slice automatically. (Your stock scanner is exactly this.) No → you're driving it by hand from a Surface.
Should the AI know about *my* projects and notes?
Yes → that's the Knowledge rail — point it at your Obsidian vault / docs via RAG, or keep a CLAUDE.md / memory file so facts persist across sessions.
